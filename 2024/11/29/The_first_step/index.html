The first step - How to Set Up a Personal Blog

DrunkDream 2024-11-29 10:56:13
Categories： Tags：

The purpose of this article is to record my method of building a personal blog. And the problems encountered will be summarized in the last part.

Ⅰ Environmental Preparation

Strongly recommend creating a new folder, and all following operations and commands should be performed within this folder. This can save a lot of trouble.

① Install Git

1. Visit the official website

Go to https://git-scm.com/, follow the prompts to install.

After installation, a prompt for Git Bash Here will appear when you right-click, and clicking on it allows you to run git commands in the current directory.

2. Run your Git terminal

Use Git Bash Here to open the terminal anywhere.

3. Check your Git version

Enter check command:

  • git –version

something similar to the following appears to indicate success:

② Install nodejs

1. Visit the official website

Enter the official website: https://www.nodejs.com.cn/download.html (Or the fastest website in your area), download according to your needs.

2. Check your installation

Enter the following command to check if the installation was successful:

  • node -v
  • npm -v

Just like:

3. Create cache and global folders

Find the directory just installed, and create two folders under the installation directory: 【node_global】and 【node_cache】.This is necessary configuration for Nodejs.

4. Configure these two folders

Use administrator privileges to open the cmd window, then input the following command:

  • npm config set prefix “your_nodejs_diretory\node_global”
  • npm config set cache “your_nodejs_diretory\node_cache”

5. Configure environment variables

The process is rather simple but too lengthy, please search it yourself XD.

③ Install Hexo

Just a few simple steps are required:

1. Execute the installation in the terminal

Enter install command and check：

  • npm install hexo -g
  • hexo -v

And you know what you should see.

2. Installation Hexo dependency

Enter the command:

  • npm install hexo-deployer-git –save

Pay special attention to the fact that this command needs to be executed in the folder where you write your blog! Otherwise, an error may occur when you upload your blog:

  • ERROR Deployer not found: git

Ⅱ Configure Your GitHub SSH Key

You should have a github account to synchronize your blog (easy to publish), and the purpose of this step is: you can upload local source code and resources to github confidentially, without having to enter your account number and password every time.

1. Enter the .ssh directory

Execute the cd command to Enter your local SSH folder:

  • cd ~/.ssh

2. Generate SSH Key

Generate SSH Key base on you email address (The one that binds to github):

  • ssh-keygen -t rsa -C “your_email_address”

Wait for a moment, then press the enter key three times consecutively.

Then, a key id_rsa.pub will appear in your .ssh directory. Execute a command in the .ssh folder:

  • clip < id_rsa.pub

to copy the key content. (Opening the text editor and then copying might cause issues!)

3. Link Start!

Open the GitHub homepage, click on personal settings.

Click on the SSH and GPG keys on the left, then click on New SSH key.

Create a title as you wish, then paste your SSH key. And just ADD IT.

Then:

Enter the command to test the connection:

  • ssh -T git@github.com

If the following prompt appears:

then you have succeeded!

Final step, configure your account and password.

  • git config –global user.name “your_github_usrname”
  • git config –global user.email “your_github-binding_email_address”

Ⅲ Build Your Personal Blog

We have finally gotten through those tedious preparation stages (if you managed to stick it out)! Now, please create an empty folder named ‘blog’, Then execute the blog initialization command under this folder:

  • hexo init

And you’ll see sth like this (Perhaps a few less documents than mine):

![blogInit](./The_first_step/blogInit.png =700x)

Now create your first blog!

  • hexo n ‘my_first_blog’

And you will find my_first_blog.md in \source_posts, just edit it as you like!

Then generate html document (This command will generate web page from all md files in your source folder):

  • hexo -g

Then start the server:

  • hexo -s

And you will have a webpage running on your local network! Click in! It might look like:

Congratulations!

Haha, you might notice that your web page theme looks different from mine? This is because I have downloaded a different theme. You can go to the Hexo Themes website to download the theme you like! Theme-related configurations are located in the \themes folder.

Ⅴ Problems

① If there is no Git Bash Here option

First, ‘win + R’ opens the run small window, and enter ‘regedit’ to open the registry.

Then, set it in the Registry editor interface:

  • HKEY_CLASSES_ROOT\Directory\Background\shell

And create a new shell folder if there is no shell folder in the Background directory.

The key points are as follows:

First, create a new item ‘Git Bush Here’ in the shell directory. At this time, click the right mouse button and the ‘Git Bush Here’ option will appear in the menu.

Then, create a new command entry under Git Bush Here, and then modify the default value of the command entry to:

  • your_git_directory\bash.exe –login -i

Get it done. At this time, you can use the Git Bash Here shortcut with confidence.

Extraly, we can add an icon to the shortcut. Create a new string value Icon under Git Bush Here and edit the Icon value:

  • your_git_directory\mingw64\share\git\git-for-windows.ico

② GitHub connection failed: Failed to connect to github.com port 443 after xxx ms: Couldn’t connect to server

When you are in a VPN environment, a GitHub connection failure is often due to inconsistent configurations between the proxy server and Git. This manifests as Git failing to access GitHub through the proxy, or the incorrect proxy port. Here are the steps to address this issue:

1. Check the current proxy settings

First, verify your system’s proxy settings. Typically, a VPN sets up a local proxy port for network requests. You can check the proxy port through the following steps:

Open Settings > Network & Internet > Proxy, find the proxy settings, and record the current proxy port. Assume the port number is xxx.

2. Configure Git proxy

Ensure that Git uses the same port as the system proxy settings. This can be configured with the following command:

  • git config –global http.proxy http://127.0.0.1:xxx
  • git config –global http.proxy https://127.0.0.1:xxx

Thus, Git will access GitHub through this proxy (including HTTP and HTTPS protocols).

3. Verify if the proxy settings have taken effect

After configuration, you can use the following command to verify if the proxy settings are correct:

  • git config –global -l

This will list the current Git configuration information, ensuring that the http.proxy and https.proxy settings within it are set to the port you just configured.

4. Refresh the DNS cache

Sometimes DNS caching can lead to connection issues. It is recommended to flush the system’s DNS cache before performing Git operations:

  • ipconfig /flushdns

Reperform Git operations to see if it can connect to GitHub normally.

③ Cannot be displayed on the web page after inserting an image in the hexo blog

1. Modify the configuration file

Set the post_asset_folder option in the _config.yml file to true to enable it.

Afterwards, when you run “hexo n ‘test’” in your command line, a ‘test’ folder will be created in the same directory as your md files. You only need to place your images in the ‘test’ folder. Then, you can reference them using relative paths (If you don’t mind the trouble, you can also create folders manually every time, make sure they have the same name!).

2. Download a plugin

Run Git Bash as an administrator in the blog directory, and execute the download command:

  • npm install https://github.com/CodeFalling/hexo-asset-image –save

And you can display images normally!
